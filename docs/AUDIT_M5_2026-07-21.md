## Narrative-neutral production evidence

`examples/m5-neutral/` is narrative-neutral, local, procedural, and offline.
Its `openai` value is the schema-required contract route namespace; generation
does not execute an OpenAI provider, model, network request, or other service.
The fixture contains real, bounded PNG/atlas, PCM WAV, TrueType, GLSL, and GLB
artifacts with specifications, requests, receipts, processing evidence,
licenses, QA, semantic bindings, and an exact path/hash/size lock.

The committed fixture contains authoring-side `production` manifests. It does
not commit a built renderpack, a built assetpack, or a finalized release
manifest. The readiness gate regenerates the fixture in disposable external
directories, builds the runtime artifacts there, verifies them, finalizes their
copied manifests, validates the resulting release profiles, and removes those
temporary outputs afterward. An authoring manifest is never itself a runtime
pack.

The temporary renderpack drives the existing 2D/2.5D standalone-game bundle,
offline replay, and package checks. The temporary `assetpack_v1` proves the M5
3D production handoff only; no current generated pyray game loads it.

## Trust boundaries

- Runtime and generated-game paths contain no provider SDK, model client,
  prompt execution, MCP configuration, credentials, weights, production
  receipts, or Forge dependency.
- Runtime inputs are finite, canonical, hash-bound, path-contained, typed, and
  independently verified before publication or import.
- The closed `assetpack_v1` schema contains runtime identity/coordinate fields,
  file records, hashes, metrics, and semantic bindings; it excludes license and
  authoring metadata. Required runtime licenses/notices travel beside the
  hash-sealed assetpack as separately verified immutable-handoff material.
- The current immutable bundle and generated pyray game accept only a
  worldpack plus `isoworld.renderpack` v1 and approved runtime files/notices.
- A 3D `assetpack_v1` is engine-neutral. M6 must select and pin an engine and
  adapter, map coordinates and semantics, and prove loading, animation,
  collision, replay, performance, packaging, and native execution in a separate
  game repository.
- Assisted authoring remains outside the runtime boundary. A route name records
  a contract namespace and never grants permission to infer or execute an
  unavailable provider capability.
