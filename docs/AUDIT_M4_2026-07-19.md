# M4 Implementation Audit — 2026-07-19

## Verdict

**PASS.** M4 establishes independent world production, immutable runtime
releases, multi-world game installation, and a clean standalone pyray/raylib
game boundary. The implementation is world-agnostic and contains no runtime AI
or dependency on a Forge/world-authoring checkout.

M4 produces a deterministic unpack-and-run source ZIP. It does not claim to
produce a wheel, installer, or native executable. Final automatic visual/audio
production belongs to M5; native wrappers require a later dedicated release
skill and target matrix.

## Adversarial findings closed

- Phase commands now bind the canonical v2 project, world identity, version,
  and strict workflow status under the same identity-safe lifecycle lock.
- P10 verifies a bounded snapshot with the runtime loader; P13 reloads that
  evidence and binds the exact worldpack hash. Reopen operations invalidate
  downstream release evidence.
- Clone and transaction paths reject symlinks, nested VCS control directories,
  credentials, unsafe parent components, and partial publication.
- Bundle, catalog, and game verifiers reject duplicate JSON keys, unexpected
  fields, path traversal, non-NFC/Windows-reserved/case-colliding paths,
  symlinks, mutable hardlinks, extra files/directories, false media types, and
  authoring/provider metadata even when an attacker recalculates public hashes.
- The generated game verifies the locked catalog implementation rather than a
  mutable wrapper, audits every project Python file, and rejects alternate
  dependency declaration surfaces.
- Runtime updates preflight every installed world and replace the entire
  snapshot atomically. They never patch individual vendored modules.
- Shared assets are permitted only below `game_data/shared/**`; their lock binds
  extension to media type and hashes `THIRD_PARTY_NOTICES.md`. G23 owns shared
  graphics, G25 owns shared audio, and other phases consume them read-only.
- The source packager includes recursive Python scripts/tests plus explicitly
  allowlisted, validated test fixtures. It snapshots once, reruns the game
  verifier in an isolated process, derives identity from that private copy,
  verifies the final archive, and publishes only to a fresh external path.
- Generated artifacts no longer expose Forge phase IDs or copy `AGENTS.md`,
  `.agents`, skills, phase evidence, editable canon, prompts, providers, model
  metadata, or Modly workflows into the game.
- G30 now includes locked Ruff lint/format checks. G32 is strictly the portable
  source-ZIP phase; native installers are not hidden inside that skill.
