# M3 implementation audit — 2026-07-19

## Outcome

M3 is complete as a deterministic, world-agnostic living-world foundation. No
blocker, high-severity, or unresolved correctness finding remains in the audited
scope. Runtime simulation consumes validated data only; it does not import or
call an AI provider.

The public contracts are worldpack v4 and save/replay v3. Worldpack v1-v3
remain loadable with empty M3 systems, while saves and replays remain bound to
the exact compiled world content hash.

- Actor needs, deterministic decay, hierarchical goal selection, schedules,

- Actor inventories, finite stockpiles, transfers, production jobs, output

- Delayed consequences, once/repeat behavior, persisted pending work, and

- Source validation, compiled loading, JSON schemas, narrative analysis,
  renderpack bindings, primitive rendering fallbacks, and compatibility paths.

## Accepted M3 boundaries

These are deliberate product boundaries, not hidden completion claims:

- The forge exposes typed `build`, `start_production`, and
  `transfer_resource` actions. A derived game owns its genre-specific build and
  economy UI.
- Construction art uses the optional semantic binding
  `construction:<blueprint_id>` and a pyray primitive fallback. Automatic asset
  inventory derivation and generation adapters belong to M5.
- M3 does not include demolition, production cancellation, market pricing,
  seasons, or inter-map goal routing.
- Goal evaluation runs at the authored world-minute boundary and can only issue
  `satisfy_need`, `travel`, `build`, or `produce`; it is not a general planner or
  script host.
- GPT, Codex, and GPT Image remain external authoring tools. Local model support
  remains external and is permitted only through Modly extensions. Neither path
  is embedded in the game runtime.

## Release recommendation

The M3 changes are ready to commit and publish. M4 can build multiple independent
world production on these stable data contracts without coupling any generated
world, roster, or lore to this repository.
