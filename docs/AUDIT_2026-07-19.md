# Repository audit — 2026-07-19

## Scope and method

The audit covered repository boundaries, source loading, validation,
compilation, worldpack loading, deterministic state updates, rendering,
agent/workflow controls, asset metadata, tests, packaging, CLI behavior, and the
M1 roadmap. The review combined code inspection, malformed-input analysis,
headless execution, reproducibility checks, runtime import auditing, and the
expanded automated suite.

This is an engineering audit of the current repository, not a penetration test
or a legal opinion on third-party assets.

## Resolved findings

| ID | Severity | Finding in M0 | M1 resolution |
|---|---|---|---|
| A-01 | High | The loader trusted the declared worldpack hash. | Recompute the canonical SHA-256 and reject mismatches. |
| A-02 | High | Schedules and abilities were accepted as data but never executed. | Typed runtime contracts, clock-driven schedules, costed abilities, and deterministic effects. |
| A-03 | High | No persistence compatibility or tamper boundary existed. | Versioned saves/replays bound to world ID and content hash, with state/final digests. |
| A-04 | High | Actors could move into the same cell. | Occupancy checks, deterministic per-tick reservations, and blocked-route recovery. |
| A-05 | Medium | Navigation was one-cell input only. | Deterministic A* routes and click-to-navigate runtime input. |
| A-06 | Medium | Map conversion had no safe, reproducible contract. | Strict finite Tiled/embedded LDtk import with explicit mappings and source/mapping hashes. |
| A-07 | Medium | Malformed cross-referenced locations could make validation brittle. | Defensive location, time, resource, effect, and route validation. |
| A-08 | Medium | Runtime content formats lacked public schemas. | Added worldpack, map, save, and replay JSON schemas. |
| A-09 | Medium | Contextual interaction was described but absent. | Range/flag/repeatability checks and atomic deterministic effects. |
| A-10 | Low | M0 UI exposed no simulation time or route state. | Clock, resources, interactions, route markers, and outcome messages in immutable render state. |

## Confirmed strengths

- Runtime and authoring packages remain one-way separated.
- Runtime contains no model/provider SDK and requires no network.
- Source and asset paths are constrained to their project roots.
- Compilation remains canonical, sorted, reproducible, and content-hashed.
- The active roster and localization remain world-defined.
- Rendering cannot mutate simulation state.
- Agent phase gates and path claims remain independent from runtime execution.
- Asset release checks retain provenance, licensing, approval, file, and hash gates.

## Accepted M1 limits and remaining risk

- Navigation is grid-local; there are no portals or inter-map route graphs.
- Reservation priority is deterministic actor ID order, not a fairness system.
- Schedule fallbacks are positional; needs, goals, and behavior trees are M3.
- Effect operations are deliberately small and allowlisted.
- Tiled/LDtk import does not compose object/entity layers or external LDtk levels.
- Save format 1 has a strict compatibility gate; no historical migration is
  needed yet, and future versions must provide explicit migrations.
- Narrative collections remain compiled data until M2 executes facts,
  relationships, dialogue, quests, and time-windowed scenes.
- Asset generation/processing adapters remain M5 work.

## Release gate

M1 is releasable only when the full headless suite passes, the neutral source
validates in release mode, its checked-in worldpack matches a clean compile, a
save and replay round trip succeeds, both map formats import in tests, and the
runtime AI-import audit reports zero findings.
